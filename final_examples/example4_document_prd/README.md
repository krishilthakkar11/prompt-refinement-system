# Example 4: Document-Heavy (PDF)

## Description
Demonstrates extraction and processing of formal requirements documents (PDFs).

## Input
PDF document containing smart home automation system requirements with structured sections.

## Expected Behavior
- Extract requirements from document structure
- High completeness from comprehensive PDF
- Source attribution to document
- Multiple deliverables

## Actual Results
- **Completeness: 1.00** (Perfect)
- **Requirements: 20** (all from document)
- **Deliverables: 7**
- **Conflicts: 0**
- **Valid: Yes**

## What It Shows
- Document extraction capabilities
- Semantic understanding (not just text extraction)
- Handles professional format (PRDs, requirements docs)
- Structure preservation from formal documents
- Source traceability for document-based requirements
